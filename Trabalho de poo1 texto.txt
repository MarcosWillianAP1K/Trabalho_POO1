Fazer um sistema de compra e estoque.

Basicamente tera tres contas, cliente, repositor e gerente.

O gerente é quem manda na merda toda, ver o historico de vendas,  solicita a compra de novos produtos para a loja,
ele consegue contratar (ou demitir) novos repositores para colocar os produtos na prateleiras, alem de consguir ver tudo oque o repositor pode ver.

O repositor é contratado pelo gerente, ele tem acesso a todo estoque e as prateleiras, seu trabalho basicamente é selecionar os 
produtos do estoque e organiza-los do jeito que bem entender nas prateleiras, consegue atribuir nome as prateleiras caso necessario,
sera registrado num historico de movimentação todas as movimentações que sairam do estoque(o mesmo acontece para entrada de produtos).

O cliente é simples, tera uma carteira pessoal (pode sacar ou depositar), e acesso a todas prateleiras no estoque, ele poderar colocar varios
itens na cesta de compra, quando finalizar a compra deverar ser debitado nas prateleiras os itens comprados, o usuario tambem terar um historico 
pessoal de compras.


Os atributos de cada conta seria basicamente
Class pessoa:
ID
Nome

metodos
ID(atribui e retorna id)

ela sera a base para as classes repositor e cliente:

class repositor:

historico_retirada_pessoal


class cliente:

historico_transações
historico_compras

class gerente:
